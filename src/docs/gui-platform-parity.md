# GUI 플랫폼 기능 비교

기준 구현은 Windows Win32 백엔드이며, Linux 백엔드는 GTK4로 구현한다. 두 백엔드는 같은 `App`, `OpenTabs`, SQLite repository, 텍스트 import/export, 검색, 찾기/바꾸기 도메인 함수를 사용한다. 메뉴와 우클릭 메뉴의 기능 목록, 활성화 정책, 키보드 단축키 scope는 `platform/gui/command_contract.rs`의 공통 명령 계약에서 관리하고, Win32는 command id, Linux는 GTK action으로 매핑한다.

## 남은 플랫폼 차이

- Windows는 native TreeView를 사용하고 Linux는 expander 버튼이 있는 GTK4 `ListBox`를 사용한다. 접기/펼치기와 주요 키보드 탐색 동작은 Win32 TreeView 기준으로 맞췄지만, 위젯의 시각적 모양은 플랫폼 기본값 차이를 따른다.
- 파일 선택과 글꼴 선택은 플랫폼 표준 위젯을 사용하므로 모양과 세부 버튼 배치는 다를 수 있다. 확인/info/error 대화상자는 GTK `MessageDialog`를 쓰지만 title/body, owner modal, Yes/No 순서, 기본 응답은 Win32 `MessageBoxW` 기준으로 보정했다. About 대화상자는 클릭 가능한 `https://github.com/edgarp9` 링크를 제공하며, Windows는 `TaskDialogIndirect` hyperlink callback, Linux는 GTK `LinkButton`으로 기본 브라우저를 연다. Linux export는 기존 일반 파일을 선택하면 앱이 Win32 저장 대화상자처럼 덮어쓰기 확인을 한 번 더 수행한다.
- Linux 창 위치는 Wayland 환경에서 앱이 강제로 복원할 수 없는 경우가 있어 현재 크기와 splitter 중심으로 복원한다.
- Windows keyboard context menu는 현재 마우스 커서 위치에 팝업을 표시한다. Linux GTK4는 Wayland 호환 전역 커서 좌표를 안정적으로 읽을 수 없어 앱 내부에서 마지막으로 관측한 트리/편집기 포인터 좌표를 우선 사용하고, 관측 좌표가 없으면 포커스 위젯 또는 선택 행 기준으로 표시한다. 마우스 우클릭 메뉴는 Windows처럼 클릭 좌표 기준으로 보정했다.

## 메뉴 항목별 실행 결과 요약

아래 표는 `platform/gui/command_contract.rs`의 현재 메뉴 계약을 기준으로 작성했다. Windows 기준 동작은 Win32 `WM_COMMAND`/native menu 구현을 기준으로 하며, Linux 검증은 실제 GTK 메뉴 클릭 smoke, D-Bus action smoke, 계약 테스트, 파일/대화상자 smoke를 함께 사용한다. Windows native GUI smoke는 CI `windows-latest`에서 실행되며, coverage guard가 모든 `GuiCommand` command id의 inline smoke 실행과 `tools/windows_menu_smoke.ps1`의 모든 `COMMAND_*` runtime 참조를 강제한다.
